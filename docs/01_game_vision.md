# 01 — 主要遊戲設計的開發願景

> **版本**：v1.0 | **日期**：2026-06-17 | **狀態**：草稿

---

## 1. 專案定位

「天堂：精煉高爽度」是一款基於**天堂 Classic MMO** 世界觀的純前端模擬遊戲，  
核心聚焦於**裝備精煉（強化）機制**的完整體驗再現。

| 面向 | 說明 |
|------|------|
| 類型 | 單人策略型裝備強化模擬器 |
| 平台 | 桌面瀏覽器（Chrome / Firefox 最新版） |
| 語系 | 正體中文（台灣） |
| 主題 | 天堂 Classic — 黑暗奇幻、中世紀城堡風格 |

---

## 2. 目標族群

- **天堂老玩家**：懷念精煉系統的刺激感，卻不想再花費真實費用
- **策略型玩家**：研究最佳精煉路線、機率期望值的玩家
- **休閒模擬愛好者**：享受即時反饋與情感波動的短時間娛樂
- **前端開發者**：以此專案學習 Vue 3 複雜互動系統的實作範例

---

## 3. 核心體驗理念

### 「精煉高爽度」三要素

```
緊張感  →  結果反饋  →  情感波動
  ↑                          ↓
  └──────── 下一次嘗試 ────────┘
```

**緊張感**  
每次精煉前明確顯示成功率，讓玩家在「衝還是不衝」之間產生決策壓力。

**即時反饋**  
成功、失敗、爆炸三種結果各有獨立動畫與音效（視覺反饋），讓結果感受清晰強烈。

**情感波動**  
裝備爆炸帶來的挫折感與高等級成功帶來的成就感，形成持續遊玩的情感驅動力。

## 4. 技術選型原則

| 原則 | 實作方式 | 理由 |
|------|---------|------|
| 零後端依賴 | 純 HTML + JS，localStorage 儲存 | 無需伺服器，任何靜態服務均可部署 |
| 輕量框架 | Vue 3 CDN 引入（`vue.global.prod.min.js`） | 無需 Node.js 建置環境，降低部署門檻 |
| 數據驅動 | `refine_rates.json` 外部配置 | 機率調整無需改動原始碼，只需更新 JSON |
| 無依賴 UI | 原生 CSS3（變數 + Flexbox + Grid） | 不引入 UI 框架，保持輕量且可控 |
| 本地優先 | Google Fonts CDN + 本地圖片資源 | 圖片本地化確保離線可用性 |

## 5. 遊戲經濟設計方向

### 天幣流動模型

```
儲值（TWD → 天幣）
        ↓
  購買卷軸（消耗天幣）
        ↓
  精煉裝備（卷軸消耗）
        ↓
  成功：裝備升值  /  失敗：裝備損失
        ↓
  出售裝備（天幣回收）
        ↓
      循環
```

### 經濟平衡原則

- **卷軸成本**固定（不受精煉等級影響），控制單次投入門檻
- **收購牌價**隨精煉等級指數成長，高等級裝備具備顯著溢價
- **玩家自訂售價**可高於牌價，模擬市場談判空間
- **初始資金 1,000 萬天幣**確保新玩家有足夠試玩空間

## 8. 未來擴充方向

- **多角色系統**：同帳號管理多個角色，各自擁有獨立背包與資產
- **排行榜**：最高精煉等級、累計爆炸次數、天幣總資產排名
- **PVP 裝備比較**：與其他玩家（localStorage 快照）比較裝備配置
- **精煉歷史記錄**：完整記錄每次精煉事件，支援回放分析
- **音效系統**：精煉成功音效、爆炸音效、背景音樂（天堂 BGM 風格）
- **暗黑模式 / 主題切換**：支援不同色彩主題（天堂、傳說、暗黑）
- **裝備命名系統**：玩家可自訂裝備名稱，增加個人化體驗
