# 場景 3D 化(2.5D 等角)設計

日期:2026-06-22
檔案:`lineage.html`(遊戲邏輯 + UI)、新增 `scene3d.js`(Babylon 呈現層)
相關決策:[ADR-0001](../../adr/0001-babylon-3d-isometric-scene.md)、[ADR-0002](../../adr/0002-rebuild-navigation-in-3d.md)
詞彙:見根目錄 `CONTEXT.md`

## 目標

把現在的 canvas 2D 俯視場景改寫成 **Babylon.js 真 3D**,以 **2.5D 等角 45°** 呈現,神似(非像素級還原)天堂經典版「古魯丁地監」地牢氛圍。設置可切換**固定視角**與**可旋轉視角**。為未來**選角系統**預留架構。

## 定案決策(grilling 共識)

| # | 主題 | 定案 |
|---|------|------|
| 1 | 還原 vs 旋轉衝突 | 旋轉優先;目標圖當**美術方向參考**,非像素還原 |
| 2 | 引擎 | **Babylon.js**(CDN ES module,無建置);因未來選角需骨架動畫/instancing |
| 3 | 投影 | **正交(Orthographic)** + 古典 2:1 等角,俯角約 30°,方位左上→右下 45° |
| 4 | 相機模式 | **固定視角**=鎖 alpha/beta;**可旋轉視角**=放開 alpha(水平繞圈)、鎖 beta(俯角永遠等角)、可縮放、平滑轉。預設**固定** |
| 5 | 邏輯層 | 導航/碰撞**在 3D 重構**(非 2D 邏輯包皮) |
| 6 | 導航底層 | **3D 原生**:射線碰撞 + 格子 A*,**不上物理引擎** |
| 7 | 地圖來源 | **單一 tilemap(2D 陣列)** 同時生 3D 牆 mesh 與 A* 導航網格 |
| 8 | 素材 | **混合**:場景程序化(方塊+`DynamicTexture`)+ 角色 CC0 GLB(骨架走路動畫) |
| 9 | 角色 | 玩家=**騎士**(預設),真 3D 模型 360° 平滑轉向 + 單一走路動畫(可選量化成 8 方向);抽象成可抽換 `Character`,未來接選角 |
| 10 | 歐林 | NPC,靜態不動,風格與場景一致 |
| 11 | 光影 | **暗環境光 + 火把/營火暖橘點光**即可;泛光/霧/粒子特效延後(視為外掛) |
| 12 | 程式結構 | WebGL canvas 在底層,Vue/HTML UI overlay 照舊;場景碼抽成 `scene3d.js` 模組 |
| 13 | 設置 | `setting.html` 加「視角模式」,存 `lineage_config.cameraMode`,預設 `'fixed'` |

## 模組邊界

```
lineage.html  ── 遊戲邏輯(座標、3D 導航、碰撞、狀態)+ 所有 UI(Vue/HTML overlay)
   │  薄介面:餵狀態進場景 / 收場景回報的地面點擊
   ▼
scene3d.js    ── 純 Babylon 呈現:讀邏輯狀態畫場景、相機、光、角色動畫、回報 ground pick
```

| 單元 | 職責 | 依賴 |
|------|------|------|
| `tilemap`(資料) | 地圖唯一真值:每格 地面/牆/柱/道具 | — |
| 牆/地面生成 | 由 tilemap 生 3D mesh | tilemap、Babylon |
| 導航網格 + A*(3D) | 由 tilemap 生可走格;x/z 平面 A* + 直線平滑 | tilemap |
| 碰撞停止 | 朝移動向量射 ray,撞牆 mesh 即停 | 牆 mesh |
| 相機 | `ArcRotateCamera` 正交;固定/可旋轉模式套用 | `cameraMode` |
| `Character` | 騎士 GLB:位置=邏輯座標映射、朝向=移動向量、播走路動畫 | 邏輯狀態 |
| NPC(歐林) | 靜態 GLB + 浮空名牌 | — |
| 光影 | 暗環境光 + 火把點光 | — |
| 點擊走路 | 地面 picking → 世界座標 → A* 終點 | A*、相機 |

## 座標映射

邏輯沿用世界座標 `(wx, wy)`(像素級),呈現時映射到 Babylon `(x, 0, z)`;比例係數在實作時定(建議 1 邏輯像素 = 固定常數單位)。相機/光/角色一律吃映射後座標。

## 場景元素驗收清單

**必須有(達標門檻):** 不規則暗色石牆迷宮(看得到牆頂+牆面)、髒污石板地面、火把數支(點亮暖光)、歐林+攤位(陶罐/小桌)+「歐林【雜貨商】」浮空名牌、騎士、中央營火/火盆。

**加分項:** 玩家腳下傳送法陣光圈、散落骨頭/陶罐/蝴蝶、石柱、地面血污斑點(現有 `drawStains` 的 3D 版)。

## 驗證

以瀏覽器預覽工具實測並截圖,跟目標圖**並排比對氛圍**(構圖、配色、必須元素到位率),非像素級比對。
1. 固定視角:啟動 → 等角 45° 地牢、必須元素到位、騎士站立、點地板自動走到目標。
2. 可旋轉視角:設置切換 → 水平繞圈、俯角不變、不穿幫。
3. 導航/碰撞:點可走處自動尋路;點牆/外提示無法到達;途中受阻立即停。
4. 走路動畫:移動時播放、朝向跟隨移動向量。

## 不做(YAGNI / 延後)

- 選角 UI、多角色、角色切換(僅預留 `Character` 抽象)。
- 物理引擎(Havok)。
- 火焰粒子、GlowLayer 泛光、霧、暗角等特效(日後外掛)。
- 像素級還原目標圖。
- 完全自由的 3D 環繞(俯角永遠鎖等角)。
