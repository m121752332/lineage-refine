# 07 — 精煉後的產品價值設計

> **版本**：v1.0 | **日期**：2026-06-17 | **狀態**：草稿

---

## 1. 價值評估框架

精煉後裝備的天幣市值由三個因素決定：

```
裝備市值 = 基礎牌價（歐林收購價）× 稀有度係數 × 市場供需
              ↓                       ↓              ↓
         系統固定設定            等級越高越稀有      玩家自訂售價
```

### 稀有度與精煉等級的關係

精煉等級越高，成功機率越低，統計上需要消耗的資源（卷軸 + 天幣 + 爆掉的裝備）越多，故市值呈**指數成長**。

---

## 2. 歐林收購牌價設計

### 資料結構

```javascript
// lineage_config.priceConfig
{
  weapon: {
    list: [0, 1000, 2000, 4000, 8000, 16000, 32000, ...],  // 歐林買入價（各等級）
    sell: [0, 0, 0, 0, 0, 0, 0, ...]                        // 玩家自訂售價（0=不自訂）
  },
  armor: {
    list: [0, 800, 1600, 3200, 6400, 12800, 25600, ...],
    sell: [0, 0, 0, 0, 0, 0, 0, ...]
  },
  twdRate: 100
}
```

- `list[n]`：`+n` 精煉等級的歐林收購牌價
- `sell[n]`：玩家設定的實際售價，0 表示使用牌價
- 實際售出價格：`Math.max(list[n], sell[n])`

## 4. 售價設定頁（setting.html）介面規格

```
┌──────────────────────────────────────────┐
│  ⚙ 歐林收購配置                          │
├──────────────────────────────────────────┤
│  最高精煉上限：[+20 ▼]                   │
│  儲值匯率：1 TWD = [100] 天幣            │
├──────────────────────────────────────────┤
│  【武器牌價設定】                         │
│  等級  歐林牌價（買入）  我的售價（選填）  │
│  +1   [______1,000]   [____________]    │
│  +2   [______2,000]   [____________]    │
│  ...                                     │
│  +20  [__2,000,000]   [____________]    │
├──────────────────────────────────────────┤
│  【防具牌價設定】                         │
│  （同上結構）                            │
├──────────────────────────────────────────┤
│  [  儲存設定  ]    [  重置為預設  ]      │
└──────────────────────────────────────────┘
```

### 欄位規格

| 欄位 | 類型 | 說明 |
|------|------|------|
| 歐林牌價 | `<input type="number">` | 系統收購參考價，可修改 |
| 我的售價 | `<input type="number">` | 留空 = 使用牌價；填入時取兩者最高 |

## 5. 出售裝備流程

```
背包選擇裝備
      ↓
計算售出價格：
  sell[enhance] > 0 → 使用玩家自訂售價
  sell[enhance] = 0 → 使用歐林牌價 list[enhance]
      ↓
顯示確認訊息：
  「出售 ⚔ 長劍 +7 → 獲得 120,000 天幣？」
      ↓
玩家確認
      ↓
背包移除裝備（設為 null）
天幣增加
訊息日誌記錄
```

## 6. 價值閉環設計

```
儲值 100,000 TWD（= 10,000,000 天幣）
              ↓
    購買 20 組祝福卷軸（假設 500,000 天幣）
              ↓
    對 +12 武器使用祝福卷軸（成功率 31%）
      成功（+13）：市值 15,000,000 天幣 ← 獲利
      失敗（爆炸）：損失 +12 裝備（6,000,000）
              ↓
    出售成功裝備，回收天幣
              ↓
    再次儲值 / 繼續精煉
```

### 期望值分析（以 +12 → +13 為例）

```
成功率：11%（普通）/ 31%（祝福）/ 51%（詛咒）

使用祝福卷軸期望：
  成功機率：31%
  成功收益：+15,000,000（+13 價值）- 6,000,000（+12 成本）= +9,000,000
  失敗機率：69%
  失敗損失：-6,000,000（+12 爆炸）

期望值 = 0.31 × 9,000,000 + 0.69 × (-6,000,000)
       = 2,790,000 - 4,140,000 = -1,350,000 天幣

→ 負期望值，但追求 +13 的爽度無法用數字衡量 😄
```

## 7. 過剩裝備的處理策略

| 情境 | 建議策略 |
|------|---------|
| 背包滿了 | 出售最低精煉等級的同類裝備 |
| 安定等級裝備（武器+5/防具+3） | 高量持有，用於批量鐵鎚精煉 |
| +6~+9 武器 | 視當前天幣決定繼續精煉或出售回本 |
| 大量 +1 裝備 | 速售清空（牌價低，不值得再精煉） |
| 詛咒卷軸用剩 | 優先對 +0 裝備使用，最大化高機率優勢 |
