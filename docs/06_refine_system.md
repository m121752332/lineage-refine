## 1. 精煉系統概覽

精煉系統是本遊戲的核心機制，允許玩家消耗卷軸與天幣，嘗試提升裝備的精煉等級（`+0` 至 `+20`）。

### 核心流程

```
選擇裝備 → 選擇卷軸（或直接強化）→ 計算成功率 → 扣除費用 → 決定結果
    ↓               ↓                    ↓              ↓
  背包選取        背包選取            即時顯示%         天幣扣除
                                                         ↓
                                                 成功 / 失敗 / 爆炸
```

---

## 2. 安定等級規格

安定等級以內的精煉成功率為 100%，無任何風險。

| 裝備類型 | 安定範圍 | 進入機率區間 |
|---------|---------|------------|
| 武器 | +0 → +5（safe_level = 6） | +6 開始 |
| 防具 | +0 → +3（safe_level = 4） | +4 開始 |

## 4. 三種卷軸行為對照

| 卷軸類型 | 成功率加成 | 成功結果 | 失敗結果 | 特殊規則 |
|---------|----------|---------|---------|---------|
| **普通卷軸** | +0% | 精煉等級 +1 | 裝備爆炸（消失） | 無 |
| **祝福卷軸** | +20% | +1 或 +2（各 50%） | 裝備爆炸（消失） | 50% 機率跳升 +2 |
| **詛咒卷軸** | +40% | 精煉等級 -1 | 裝備**必爆炸**（消失） | 成功也只降 1 級，失敗 100% 爆炸 |

### 詛咒卷軸使用情境

- 適用於 +0 裝備（失敗爆炸但無損失）
- 高機率快速升至 +1（+40% 加成）
- 不建議用於 +1 以上裝備（成功降為 +0）

---

## 5. 直接強化（不使用卷軸）

不使用卷軸的精煉行為：

- 使用**普通卷軸基礎成功率**（無加成）
- 費用較低（無卷軸購買成本）
- 失敗同樣會導致裝備爆炸

## 6. 強化費用計算

### 費用公式

```javascript
// 基礎費用隨精煉等級遞增
function getEnhanceCost(currentLevel) {
  const baseCost = 500;
  return baseCost * Math.pow(2, currentLevel);
}

// 範例：
// +0 → +1：500 天幣
// +5 → +6：16,000 天幣
// +10 → +11：512,000 天幣
// +19 → +20：262,144,000 天幣
```

## 7. 結果動畫規格

### 成功動畫

```
1. 裝備格子閃爍金色光環
2. 浮起文字：「✨ +{newLevel}」（金色，向上漂浮 + 淡出）
3. 結果彈窗：綠色背景「強化成功！」
4. 彩色粒子：黃金色粒子從彈窗散落
```

### 失敗動畫（安定等級以外，不爆炸的情況）

> 注意：本系統設計中，機率區間失敗 = 爆炸，無「失敗但不爆炸」選項。

### 爆炸動畫

```
1. 裝備格子縮放 → 旋轉 → 消失
2. 爆炸特效：「💥」向外擴散（scale + opacity 動畫）
3. 浮起文字：「💨 爆炸！」（紅色，向上漂浮）
4. 結果彈窗：紅色背景「裝備爆炸了！」
5. 訊息日誌記錄爆炸事件
```

## 8. JSON 配置格式

機率調整無需修改程式碼，只需更新 `refine_rates.json`：

```json
{
  "_note": "成功率以百分比儲存（100 = 100%），支援 6 位小數",
  "scroll_bonus": {
    "normal": 0,
    "blessed": 20,
    "cursed": 40
  },
  "weapon": {
    "name": "武器",
    "safe_level": 6,
    "rates": [
      { "from": 6, "to": 7, "base": 77.000000, "safe": false }
    ]
  }
}
```

### 新增精煉等級步驟

1. 在 `rates[]` 陣列新增對應的 `{from, to, base, safe}` 物件
2. 更新 `maxEnhLevel` 上限配置（`setting.html`）
3. 重新載入頁面即生效

## 9. 最高精煉上限配置

玩家（或管理者）可於歐林配置頁調整最高精煉等級：

```
可選範圍：+10 ~ +20
預設值：+20
儲存位置：lineage_config.maxEnhLevel
影響範圍：
  - 精煉面板不顯示超過上限的等級
  - 機率表截止顯示至上限
  - 牌價表截止至上限
```

## 11. 整批衝裝工具（2026-07 新增，見 docs/adr/0003）

背包工具列勾選「整批衝裝」（預設不勾）後，點選武器／防具卷軸改開三階段視窗（不勾時單把流程不變）：

```
階段 1｜待處理：載入背包中所有同類裝備（武器卷軸→武器、防具卷軸→防具）
        5 欄格狀清單、固定 5×5 視口可捲動；點選 toggle、SHIFT+點選範圍全選
階段 2｜指定裝備等級：下拉 +1 ~ maxEnhLevel
階段 3｜衝裝處理：「開始」後已選裝備移出背包載入 5×5 處理格，
        由左至右、由上至下輪巡，每輪每件精煉一次（+1），500ms/次拉霸式高亮
```

規則：

| 項目 | 整批衝裝 | 單把精煉 |
|------|----------|----------|
| 成功機率 | 同（複用 isSafe/successRate） | — |
| 卷軸消耗 | 每次嘗試 1 張同變體卷軸；用盡即中止 | 每次 1 張 |
| 強化費用 | **免費** | 500 × 2^當前等級 |
| 詛咒卷軸 | 不可用（成功會降級，落回單把流程） | 可用 |
| 祝福 +2 | clamp 至目標等級 | clamp 至 maxEnhLevel |
| 失敗爆裝 | 化為粉末（格內顯示 💨） | 同 |

已達目標等級的裝備載入時直接視為通過（0 次嘗試）。執行中視窗不可關閉（✕／Esc 均擋下）。
結束顯示總結（N 件經過精煉、X 件通過、Y 件化為粉末、卷軸用盡時另計未完成），
「回到背包」把存活裝備放回空格；空格不足時視窗保持開啟避免裝備遺失。
