# 04 — 儲值系統的設計

> **版本**：v1.0 | **日期**：2026-06-17 | **狀態**：草稿

---

## 1. 系統目標

儲值系統（`cash_pay.html`）模擬真實遊戲的儲值體驗，讓玩家可以將虛擬「新台幣」兌換成遊戲內貨幣「天幣」。

**重要聲明**：本系統為純模擬設計，**不涉及任何真實金流或支付行為**，所有金額操作僅在 localStorage 中進行。

---

## 2. 匯率設計

### 基礎匯率

```
新台幣（TWD）× twdRate = 天幣（Gold）

預設：1 TWD = 100 天幣
範例：100 TWD → 10,000 天幣
     1,000 TWD → 100,000 天幣
     10,000 TWD → 1,000,000 天幣
```

### 匯率配置

匯率 `twdRate` 儲存於 `lineage_config.priceConfig.twdRate`，可透過歐林配置頁（`setting.html`）調整。

| 配置值 | 換算結果 |
|-------|---------|
| `twdRate: 50` | 1 TWD = 50 天幣 |
| `twdRate: 100`（預設） | 1 TWD = 100 天幣 |
| `twdRate: 200` | 1 TWD = 200 天幣 |

## 3. 儲值金額設計

### 快速預設按鈕

提供 10 組快速選擇按鈕，覆蓋常見儲值金額：

```
┌───────┬───────┬───────┬───────┬───────┐
│ 100元 │ 300元 │ 500元 │1,000元│3,000元│
├───────┼───────┼───────┼───────┼───────┤
│5,000元│10,000 │30,000 │50,000 │100,000│
└───────┴───────┴───────┴───────┴───────┘
```

點擊後自動填入自訂金額欄位並即時計算天幣數量。

### 自訂金額輸入

```
欄位類型：<input type="number">
最小值：1 TWD
最大值：100,000 TWD（10 萬元上限）
步進：1
即時計算：輸入時同步更新「可獲得天幣」顯示
```

### 上限規格

| 限制項目 | 限制值 | 說明 |
|---------|--------|------|
| 單次儲值上限 | 100,000 TWD | 防止誤觸超額 |
| 對應天幣上限 | 10,000,000 天幣 | 1,000 萬天幣 |
| 儲值後天幣上限 | 無上限 | 累計天幣無上限 |

## 5. 互動規格

### 快速按鈕選中狀態

```css
.quick-btn.selected {
  background: var(--gold);
  color: var(--dark);
  box-shadow: 0 0 8px rgba(200, 168, 75, 0.5);
}
```

選中某個快速按鈕後，自訂輸入欄位同步更新為該金額。  
修改自訂金額後，快速按鈕選中狀態自動取消。

### 即時計算

```javascript
// 輸入框 oninput 事件
twdAmount = parseInt(input.value) || 0
goldAmount = twdAmount * twdRate
previewText = `可獲得天幣：💰 ${goldAmount.toLocaleString()} 天幣`
```

## 6. 儲值確認流程

```
點擊「確認儲值」
       ↓
驗證金額：
  - 金額 ≤ 0 → 提示「請輸入有效金額」
  - 金額 > 100,000 → 提示「超過單次上限 10 萬元」
       ↓
計算天幣：gold += twdAmount × twdRate
       ↓
更新 localStorage['lineage_player'].gold
       ↓
顯示確認彈窗：
  「儲值成功！獲得 X 天幣
   目前餘額：Y 天幣」
       ↓
確認後自動返回 lineage-refine.html
```
